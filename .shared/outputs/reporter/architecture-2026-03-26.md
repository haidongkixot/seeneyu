## 7. Key Architecture Decisions

1. **YouTube IFrame only** — never self-host video. Store `youtubeVideoId` + timestamps.
2. **Client-side ML** — MediaPipe runs in browser for all scoring. Zero API cost for Arcade + Micro.
3. **GPT-4o TEXT fallback** — Full Performance recording can optionally use GPT-4o for richer feedback.
4. **File-based team coordination** — `.shared/` directory with signals, state, memory for multi-agent development.
5. **Prisma + Neon** — PostgreSQL with connection pooling. `(prisma as any)` cast pattern when generate can't run locally.
6. **Registration approval gate** — New users start as `pending`, admin must approve before login works.
7. **Flat comment threading** — 1 level of replies only (not deep threading).
8. **Plan-based feature gating** — Basic (free, 5s), Standard (30s, voice), Advanced (3min, VIP, coach).
9. **Embeddable mini-games** — iframe-friendly routes under `/embed/games/` with PostMessage bridge.
10. **Expo SDK 52** — Mobile app uses SDK 52 for Expo Go compatibility (SDK 53 requires dev client).
